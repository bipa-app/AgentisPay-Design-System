04 — Voz & Tom

Como a Agentis Pay fala

A voz é filosófica, declarativa e precisa. Mescla referências clássicas com linguagem técnica sem jamais ser casual ou excessiva. Cada frase carrega peso. Sem exclamações. Sem rodeios. Sem amigabilidade forçada.

01
Filosófico
Eleva o discurso. Usa conceitos como "soberano", "consentimento", "vontade". Não é um app de pagamento — é infraestrutura para a era da IA.
02
Declarativo
Frases curtas e definitivas. "O dinheiro só sai com sua biometria." Sem condicionais desnecessários. A marca sabe o que é.
03
Técnico
Usa terminologia real: "trilho de liquidação", "consentimento biométrico", "ledger", "proposta". Respeita a inteligência de quem integra.
04
Clássico
Referências à Ágora grega, a Hermes, ao soberano. Posiciona tecnologia de ponta em linguagem duradoura.
05
Binário
Contraste como estrutura: máquina propõe / humano dispõe. O jeito antigo / o jeito Ágora. Sempre dois lados, sempre claro.

Vocabulário da marca

UseEvitePor quê
PropostaSolicitação / pedidoO agente propõe — o humano decide. Hierarquia clara.
O soberano dispõeO usuário aprovaReforça a autoridade humana como princípio, não feature.
Consentimento biométricoConfirmação / autenticaçãoBiometria é prova criptográfica de vontade humana.
Trilho de liquidaçãoMétodo de pagamentoPIX é um trilho — a linguagem técnica é intencional.

05 — Ilustrações

Estilo dot-matrix

As ilustrações da Agentis Pay são renderizadas em dot-matrix — formas construídas com matrizes densas de pontos e vírgulas, criando volume e textura sem linhas sólidas. O resultado é técnico, orgânico e inconfundível. Fundo sempre preto. Variações em laranja --orange e branco.

Símbolo abstrato — Agentis

Losango de quatro pétalas. Remete ao símbolo PIX e à interseção entre agente e humano.

Branco · #FFFFFF

Regras de uso

✓ Texto de interface

Use --t, --t2 ou --t3-accessible. Nunca use --t3 (#4A6070) para texto legível — ratio 2.88:1 falha WCAG.

✓ Botão primário (label)

Para texto bold ≥ 14px em fundo laranja: --orange é AA Large (ok). Para texto regular ou menor: use fundo --orange-dark (#CC4400).

✓ Mensagens de erro

Use --danger-fg (#E05040, 4.86:1) para texto de erro legível. Reserve --danger (#C03A2B) apenas para ícones, bordas e fundos.

⚠ Teal como texto

--teal tem ratio 3.78:1 em fundo escuro — não usar como texto normal. OK para ícones, bordas e elementos decorativos grandes.

✓ Focus ring

Use outline: 2px solid var(--orange); outline-offset: 2px em todos os elementos interativos. Orange sobre bg escuro tem 5.90:1 — passa AA.

✓ Tamanho mínimo de toque (mobile)

WCAG 2.5.5 recomenda área de toque mínima de 44×44px. Os botões mobile já usam altura 48px — manter esse padrão em todos os elementos clicáveis.

Token set completo para tema claro. Aplicado via data-theme="light" no elemento raiz. Todos os tokens de cor, borda e sombra são sobrescritos — brand (orange/teal), tipografia, espaçamento e radius permanecem inalterados.

Como ativar

HTML
<body data-theme="light">
  ...
</body>
JavaScript
document.body

[data-theme="light"] {
  --color-bg-base: #F5F7FA;
  /* ... demais tokens */
}

Regras de uso no Light Mode

✓ Brand permanece laranja

Os tokens --color-brand-orange e --color-brand-teal são idênticos nos dois modos. Botões, badges e focus rings não mudam.

✓ Semântico mais saturado no light

Success (#059669) e Danger (#DC2626) são versões mais escuras que passam AA em fundo branco. Não usar os valores dark (#34D399, #C03A2B) sobre #FFFFFF.

✓ Sombras reduzidas

No light mode, sombras usam opacidade 0.06–0.16 (vs 0.40–0.80 no dark). Sobre fundo branco, sombras pesadas parecem artificiais — os novos valores ficam mais naturais.

⚠ Tipografia e radius não mudam

Space Grotesk, Space Mono, todos os valores de --radius-*, --spacing-* e --motion-* são idênticos nos dois modos. Não sobrescrever.

✓ Storybook com toolbar 🌙/☀️

O Storybook expõe um seletor de tema na toolbar. Ao trocar, aplica data-theme="light" no body e troca o fundo para #F5F7FA automaticamente.

✓ Preferência do sistema

Para respeitar prefers-color-scheme: detecte via window.matchMedia e aplique o atributo. O CSS já estará pronto para responder.
